# PartSearchSuggest ‚Äî v0.8.0.0 Thumbnail Experiments Plan

> **Status: DEFERRED / LIKELY ABANDONED (2026-07-07)**  
> Thumbnails worsen VAB load and index time on large mod lists (~150 mods). **v0.8.x goal is search-term expansion** (CCK tabs, subassemblies) per [`ROADMAP.md`](ROADMAP.md). This document is kept for historical reference only.

**Document version:** 2026-07-07  
**Mod:** Part Search Suggest (`PartSearchSuggest`)  
**Track:** v0.8.0.0 ‚Äî thumbnail experiment line (separate from v0.7.x stability work)  
**Prerequisite stable line:** v0.7.6 (race fixes); v0.7.7 (editor part availability / asteroids) may land first  
**Gate before v0.9.0 limited rollout:** text-only dropdown must remain shippable; thumbnails are experimental until Phase 4 sign-off

## Why v0.8.0.0 exists

The dropdown is **text-only and stable** as of v0.7.6. Thumbnails were attempted early in the POC (v0.2), removed in v0.4, and deferred while ghosting, layout, scroll, and search quality were fixed through v0.7.x.

v0.8.0.0 **parses thumbnail work into isolated experiment phases** so each spike can ship (or fail) without blocking the v0.9.0 rollout path.

## Phase 0 ‚Äî Research (v0.8.0.0, plan-only)

**Goal:** Document how KSP renders editor part icons and which approaches are viable for a 24√ó24 dropdown slot.

### 0.1 How KSP renders part icons (EditorPartList)

Stock pipeline (KSP 1.12.x, from API docs + assembly references):

| Component | Role |
|-----------|------|
| `AvailablePart.iconPrefab` | `GameObject` prefab reference for the part's editor icon (3D mesh renderers, not a flat sprite) |
| `AvailablePart.iconRenderers` | Cached `Renderer[]` on `iconPrefab` |
| `EditorPartList` | Owns `partGrid`, `iconSize`, `iconOverScale`, `iconOverSpin`, and `EditorPartIcon partPrefab` template |
| `EditorPartIcon` | Stock UI wrapper; `Create(EditorPartList, AvailablePart, iconSize, ‚Ä¶)` builds a row icon; exposes `partIcon` ‚Üí `KSP.UI.Screens.Editor.PartIcon` |
| `PartIcon` | Holds `Icon` (`GameObject`) + cached `Renderers`; `DestroyIcon()` for cleanup |
| `EditorPartIcon.inventoryItemThumbnail` | `RawImage` used when `useImageThumbnail: true` (2D path, e.g. inventory) |
| `PartLoader.getPartInfoByIconPrefab` | Reverse lookup from cloned icon ‚Üí `AvailablePart` |

**Render path:** Editor icons are **3D objects** rendered into the UI canvas hierarchy (meshes + materials), not pre-baked `Sprite`s on `AvailablePart`. Variants, flags, and grey-out states are handled by `EditorPartIcon` material helpers (`SetPartColor`, `CreateMaterialArray`, `CleanUpMaterials`).

**Research tasks (no plugin changes required):**

1. Use `tools/ReflectSearch/` or ILSpy on `Assembly-CSharp.dll` to trace `EditorPartIcon.Create` ‚Üí icon instantiation for a known part (e.g. `liquidEngine`).
2. Log `EditorLogic.fetch.partList.iconSize` / `iconOverScale` at editor load ‚Äî match dropdown thumbnail scale to stock for visual consistency.
3. Determine whether `useImageThumbnail` + `inventoryItemThumbnail` can be reused for dropdown rows (lighter than full 3D clone).
4. Check interaction with **KSPCommunityFixes `FasterEditorPartList`** (ModTest has it on main install) ‚Äî icon pool / refresh timing may differ.
5. Document cleanup: `PartIcon.DestroyIcon()`, `EditorPartIcon.CleanUpMaterials()` ‚Äî required on row destroy to avoid material leaks.

### 0.2 What failed in the early POC

| Issue | What happened | Root cause (best understanding) |
|-------|---------------|----------------------------------|
| **"Thumbnails" in v0.1** | User saw part images in/near dropdown | **No thumbnails existed** ‚Äî semi-transparent panel + parts list visible behind rows looked like row icons |
| **v0.2 thumbnail spike** | Text separated from image; clipped at panel edge | Horizontal layout not integrated with row `LayoutElement`; label still `StretchFull` centered over entire row |
| **Ghosting (v0.4.x)** | VAB part icons visible **through/on top of** dropdown text | **Draw order**, not alpha ‚Äî `scrollListSub` icons are later siblings in the same canvas; opaque `Image` alone insufficient |
| **Canvas layering fixes (v0.4.5‚Äì0.4.6)** | Dropdown **invisible** | Nested `Canvas` + `overrideSorting` breaks rendering in KSP editor UI (repeat regression) |
| **Click pass-through (v0.4.4)** | Clicks hit parts beneath dropdown | Fixed with raycast targets + dim blocker; separate from ghosting |
| **PartThumbnailHelper removed (v0.4)** | Thumbnail code deleted | User directive: stabilize text-first; helper unused after layout failures |

**Current ghosting mitigation (v0.5+):** slide/hide parts catalog via `PartsPanelCollapseHelper` while dropdown is open. Thumbnail experiments must **not** assume catalog is visible ‚Äî but row icons must still render correctly when catalog is off-screen.

### 0.3 Candidate approaches (ranked for Phase 1)

| # | Approach | Pros | Cons |
|---|----------|------|------|
| A | **Clone `AvailablePart.iconPrefab`** into a fixed `RectMask2D` slot (v0.2 retry) | Simple; no `EditorPartList` dependency | Scale/camera/lighting may wrong; 3D in UI layout fragile |
| B | **`EditorPartIcon.Create`** with reference `EditorPartList`, strip buttons/interaction | Stock-correct appearance, variants | Heavy; needs live `EditorPartList`; harder to pool |
| C | **`useImageThumbnail` path** via `EditorPartIcon` overload | Potentially cheaper 2D `RawImage` | May not exist for all parts; needs verification |
| D | **Snapshot to `RenderTexture` ‚Üí `RawImage`** (custom cache) | Stable 2D UI; good for scroll perf | Up-front render cost; cache invalidation on variants |

**Phase 1 default:** try **A** first (minimal spike); fall back to **C** or **D** if A fails visually or on performance.

## Phase 1 ‚Äî Minimal spike (v0.8.0.0-a)

**Goal:** One part row shows a 24√ó24 thumbnail beside text; no scroll/layout regression.

### Scope

- **Part suggestions only** (`SuggestionKind.Part` && `Part != null`)
- First visible row only, OR all part rows ‚Äî start with **first row only** behind `#define THUMBNAIL_EXPERIMENTS` / config flag to limit blast radius
- Fixed icon size: **24√ó24** px (tunable constant `ThumbnailSize = 24f`)
- Text-only rows unchanged for metadata/history/categorizer

### Acceptance criteria

## Phase 2 ‚Äî Layout (v0.8.0.0-b)

**Goal:** Row height, padding, and text alignment work with thumbnail for all part rows.

### Tasks

- Row height: bump single-line from 30px ‚Üí **36px** (or 38px) when thumbnail enabled; keep 42px subtitle variant proportional
- **Left-align** title + subtitle; reserve left gutter = `ThumbnailSize + padding`
- Icon vertical centering in row via `HorizontalLayoutGroup` + `LayoutElement`
- **Opaque row backgrounds** ‚Äî retain alpha 1.0 on row `Image`; icon slot gets solid backing color matching row (prevents mesh bleed)
- Subtitle (`MatchReason`) stays on second line, aligned with title text column (not under icon)
- Header/panel padding audit ‚Äî ensure thumbnail column does not clip at panel edges

### Acceptance criteria

- [ ] 10+ part results: no horizontal clip, no text/icon overlap
- [ ] Subtitle rows readable (`category:engine`, etc.)
- [ ] No ghosting of **catalog** icons through rows (catalog still collapsed)
- [ ] No ghosting of **row** icon meshes through text (RectMask2D + opaque backing)

## Phase 3 ‚Äî Performance (v0.8.0.0-c)

**Goal:** Lazy load, cache, and cap concurrent icon builds so scroll + many rows stays responsive.

### Tasks

- **Lazy load:** create icon only when row is built; defer off-screen rows until scroll brings them near viewport (optional Phase 3b)
- **Cache:** `Dictionary<string, Sprite or RawImage texture>` keyed by `part.name` (+ variant id if applicable)
- **Concurrency cap:** max N icon instantiations per frame (e.g. 2‚Äì4) ‚Äî queue remainder
- **Cleanup:** on `ClearRows()`, destroy cloned icons + call stock cleanup helpers; release cache entries on editor exit
- **Stress test:** query `engine` or `tank` on heavy install (~118 mod folders) ‚Äî 24 suggestions, scroll full list
- **Interaction with FasterEditorPartList:** verify no double-pooling or stale icons

### Metrics (log or stopwatch)

- Time from `Show()` to first frame with all visible thumbnails
- Frame spike count when opening dropdown with 24 part rows
- Memory: cache size after 50 unique queries in one session

### Acceptance criteria

- [ ] Opening dropdown with 24 part rows: no multi-second freeze
- [ ] Scrolling 100+ row session (repeated queries): no obvious leak in 30-min editor session
- [ ] Cache hit on repeated query returns instantly

## Phase 4 ‚Äî Scope & feature flag (v0.8.0.0-d)

**Goal:** Decide production scope; allow disabling thumbnails without rebuild.

### Scope decisions

| Row kind | Thumbnail? | Notes |
|----------|------------|-------|
| `SuggestionKind.Part` | **Yes** (default) | Primary value |
| `SuggestionKind.History` | No | Text-only query replay |
| Metadata (author/mod/suite) | **No** (v1 of thumbnails) | No single `AvailablePart`; optional future "mod icon" is out of scope |
| Categorizer filters | **No** (v1) | Could show generic category glyph later ‚Äî defer |
| First-class metadata tint | Preserve | Thumbnail column only on part rows |

### Feature flag options (pick one in implementation)

1. **Compile-time:** `#define THUMBNAIL_EXPERIMENTS` in csproj (dev builds only)
2. **Runtime config:** `GameData/PartSearchSuggest/PluginData/Settings.cfg` ‚Üí `enableThumbnails = false` default until sign-off
3. **Both:** compile gate for experimental builds; runtime flag for users

**Recommendation:** runtime default **off** until v0.8.0.0-d passes QA; enable on ModTest for development.

### Phase 4 exit criteria (ready to *consider* for v0.9.x)

## Risks & mitigations

| Risk | Severity | Mitigation |
|------|----------|------------|
| **Z-order / ghosting** (catalog icons over dropdown) | High | Keep `PartsPanelCollapseHelper` slide; do not rely on canvas `overrideSorting` (broke v0.4.5‚Äì0.4.6) |
| **Row icon ghosting** (3D mesh through text) | Medium | `RectMask2D` on icon slot; opaque row backing; prefer 2D snapshot path if mesh bleed persists |
| **Click pass-through** | High | Preserve row `Image.raycastTarget = true`, `PointerDown` handler, dim blocker; icon child `raycastTarget = false` |
| **Layout regression** | Medium | Horizontal layout + explicit `LayoutElement` sizes; never `StretchFull` label over icon column |
| **Performance / editor hitch** | High | Phase 3 cache + lazy load + per-frame cap; stress test on heavy install |
| **Material / GameObject leak** | Medium | Destroy icons in `ClearRows()`; use stock `CleanUpMaterials` / `PartIcon.DestroyIcon` |
| **Variant / TweakScale / B9PS icons wrong** | Low‚ÄìMed | Cache key includes variant; test B9PartSwitch parts on main install |
| **FasterEditorPartList interaction** | Low | Test with KSPCommunityFixes enabled on main profile |
| **Thumbnail scope creep** | Low | Phase 4 table ‚Äî part rows only for v1 |
